Entwicklung einer MVC-Webanwendung

Übungsaufgabe „Wastebin” 🗑️

Nachdem du so tapfer die ersten beiden Foliensätze zu „Grundlagen verteilter Systeme” und „Enterprise Java Beans” durchgeackert hast, wollen wir diese Mühe gerne belohnen. 🏆 Deshalb haben wir hier eine kleine Aufgabe für dich, bei der du die wichtigsten Aspekte von MVC mit Servlets, JSP und EJB nochmal ausprobieren kannst.

Unsere kleine Anwendung heißt „Wastebin” und hat nur rein zufällig etwas mit dem ähnlich klingenden Plagiat Pastebin zu tun. 🤞🏼 Ähnlich wie dort soll unsere Anwendung dazu genutzt werden, Codeschnippsel zu speichern. Die folgenden Screenshots zeigen, wie die Anwendung aussieht:

Der Quellcode ist wie folgt strukturiert:

Fachliche Klassen

  • Paket ….wastebin.jpa
    Klasse Waste
    Persistence Entity für einen Codeschnippsel
  • Paket ….wastebin.jpa
    Klasse WasteType
    Aufzählung der Programmiersprachen für das Feld "Typ"
  • Paket ….wastebin.ejb
    Klasse WasteBean
    EJB mit Funktionen zum Anlegen, Lesen, Ändern und Löschen von Einträgen

Weboberfläche

  • Paket ….wastebin.web
    Klasse IndexServlet
    Servlet für die Startseite
  • Datei /WEB-INF/index.jsp
    JSP für die Startseite
  • Paket ….wastebin.web
    Klasse CreateServlet
    Servlet für das Anlageformular
  • Datei /WEB-INF/form.jsp
    Formular zum Anlegen eines Eintrags
  • Paket ….wastebin.web
    Klasse WasteForm
    Hilfsklasse zum Prüfen der Formulareingaben
  • Paket ….wastebin.web
    Klasse ViewServlet
    Servlet für die Anzeige eines Eintrags
  • Datei /WEB-INF/view.jsp
    JSP zum Anzeigen eines Eintrags
  • Datei /WEB-INF/tags/templates/base.tag
    HTML-Template für alle Seiten
  • Datei style.css
    CSS-Stylesheet
  • Ordner /prism/
    Prism.js: JavaScript-Bibliothek für das Syntax Highlighting

Aufgabenstellung

  1. Lade dir von Moodle die ZIP-Datei Wastebin_Aufgabe.zip herunter und importiere sie in NetBeans.
  2. Vervollständige die Methode doGet() der Klasse IndexServlet.
  3. Vervollständige die Methode doPost() der Klasse CreateServlet.
  4. Vervollständige die Datei /WEB-INF/form.jsp, so dass die Fehlermeldungen ausgegeben werden.
  5. Vervollständige die Methode doGet() der Klasse ViewServlet.

Die anzupassenden Stellen sind mit „TODO” und einer kurzen Erklärung gekennzeichnet. Hauptsächlich geht es darum, die EJB an der richtigen Stelle aufzurufen und ein wenig dein Wissen aus Webprogrammierung anzuwenden. 🙂
